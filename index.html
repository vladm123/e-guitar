e-guitar

freelance progress tracker

Download this project as a .zip file Download this project as a tar.gz file

User stories

  1. As a freelancer I want to be able to register the time I spend on my projects, so that I can document time spent for my customers.
  2. As a freelancer I want to be able to get an overview of my time registrations, so that I know how much I have worked.

Overview

Freelancers are now able to register intervals of time in which they worked for specific projects. They can share their progress and working hours to the customers via permanent links. Freelancers can get an overview of their time spent for each task.

Stack

  • Database: mongodb
  • Backend: node, express
  • Frontend: jQuery
  • Test: mocha, should

UI

GET / Shows information related to all projects, which allow:

  • add: select the add option from the project manager
  • modify: click the project title and select the change option from the project manager
  • remove: click the project title and select the remove option from the project manager

Each project can contain zero or more tasks, which can be:

  • start -> work in progress: click the + option from the project, which will make it orange and empty
  • work in progress -> complete: click the project circle, which will change it to green and full

GET /{identifier} Shows read-only information related to a specific project. This can be easily shared with customers.

API

Projects

List

GET /projects Selects all projects, along with the tasks for each project. Returns the following information:

  • 200, along with the selected projects in JSON format, if the request was successful.

Select

GET /projects/{identifier} Selects the project by identifier. The identifier must be a 24-character string of hexadecimals. Returns the following information:

  • 200, along with the selected project in JSON format, if the request was successful.
  • 400, if the identifier was invalid.
  • 404, if the project was not found.

Insert

POST /projects/insert {"name": <name>[, "description": <descripion>]} Inserts the project specified in the body in JSON format. The name must be a none-mpty string. Returns the following information:

  • 201, along with the project URL in the header location and the inserted project in JSON format, if the request was successful.
  • 400, if the name was not defined or was the empty string.
  • 404, if the project was not found.

Update

POST /projects/{identifier}/update {"name": <name>[, "description": <descripion>]} Updates the project by identifier specified in the body in JSON format. The identifier must be a 24-character string of hexadecimals. The name must be a non-empty string. Returns the following information:

  • 200, along with the updated project in JSON format, if the request was successful.
  • 400, if the identifier was invalid.
  • 400, if the name was not defined or was the empty string.
  • 404, if the project was not found.

Delete

POST /projects/{identifier}/delete `` Deletes the project by identifier. The identifier must be a 24-character string of hexadecimals. Returns the following information:

  • 200, along with the deleted project in JSON format, if the request was successful.
  • 400, if the identifier was invalid.
  • 404, if the project was not found.

Tasks

List

The tasks are listed as part of each project. There is no need for an API method to expose them on their own.

Select

The tasks are listed as part of each project. There is no need for an API method to expose them on their own.

Insert

POST /projects/{identifier}/tasks/insert {["start": <start>[, "end": <end>]]} Updates the project by identifier by inserting the task specified in the body in JSON format. The identifier must be a 24-character string of hexadecimals. The start must be a long number representing the epoch time, if exists. The start is populated with the current epoch time, if not exists. The end must be a long number representing the epoch time, if exists. Returns the following information:

  • 201, along with the project URL in the header location and the updated project in JSON format, if the request was successful.
  • 400, if the identifier was invalid.
  • 400, if the start was invalid.
  • 400, if the end was invalid.
  • 404, if the project was not found.

Update

POST /projects/{identifier}/tasks/update {"start": <start>[, "end": <end>]} Updates the project by identifier by updating the task specified in the body in JSON format. The tasks that have the same start date as the provided start date are updated. The identifier must be a 24-character string of hexadecimals. The start must be a long number representing the epoch time. The end must be a long number representing the epoch time, if exists. Returns the following information:

  • 200, along with the updated project in JSON format, if the request was successful.
  • 400, if the identifier was invalid.
  • 400, if the start was missing.
  • 400, if the start was invalid.
  • 400, if the end was invalid.
  • 404, if the project was not found.

Delete

POST /projects/{identifier}/tasks/update {"start": <start>} Updates the project by identifier by deleting the task specified in the body in JSON format. The tasks that have the same start date as the provided start date are deleted. The identifier must be a 24-character string of hexadecimals. The start must be a long number representing the epoch time. Returns the following information:

  • 200, along with the updated project in JSON format, if the request was successful.
  • 400, if the identifier was invalid.
  • 400, if the start was missing.
  • 400, if the start was invalid.
  • 404, if the project was not found.

Extensions

  • Client code is written in pure jQuery. Some methods might be split and encapsulated.
  • Permanent link identifier coincides with the project identifier. This can be changed to a secret string.
  • Added extensive automation for project selection. Same pattern can be applied to the others.
  • Coupled the model to the mongo. Another layer can be implemented so that database is abstracted out.
